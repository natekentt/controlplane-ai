# Skill Meta-Template

All skills follow this structure. Each section is required unless marked optional.

---

## Purpose

One or two sentences describing when this skill applies. Agents use this to decide whether to load the skill (lazy loading).

## Rules

Numbered list of opinionated defaults. Number rules so they can be referenced deterministically (e.g., "per Rule 3").

1. Rule statement.
2. Rule statement.

Keep to ~10 rules. Enough to be useful, few enough to be memorable.

## Examples

Concrete Good vs Bad patterns. Use language-agnostic pseudocode unless the skill is language-specific.

### Good

```
example
```

### Bad

```
example — with brief explanation of why
```

## Exceptions *(optional)*

When it's acceptable to break the rules. Each exception should reference the rule number it overrides.

- **Exception to Rule N**: Rationale.

Links to related skills, templates, or external standards.
